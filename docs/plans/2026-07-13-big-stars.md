# Big Stars Size-Gating Plan

This plan implements big stars (dots) that require Eater to be a certain size before they can be eaten, acting as a progression puzzle.

## 🏛️ Architecture & Code Changes

### 1. Dot Data Structure
- Add a boolean flag `big` and custom `radius` (6 for small, 12 for big) to the dot objects:
  - Small: `{ x: ..., y: ..., collected: false }` (defaults to small, radius 6)
  - Big: `{ x: ..., y: ..., collected: false, big: true, radius: 12 }`

### 2. Level Design Updates
- **Level 1**:
  - Make the highest dot at `(610, 120)` big:
    `{ x: 610, y: 120, collected: false, big: true }`
- **Level 2**:
  - Make the highest dot at `(520, 110)` big:
    `{ x: 520, y: 110, collected: false, big: true }`

### 3. Collection Logic
- During the collectibles loop in `update()`, check the size gating:
  - A big dot has `dot.big === true`.
  - Eater's current size is `player.radius`.
  - Gating condition: if a dot is big, the player must have `player.radius >= 18` to eat it.
  - If `player.radius < 18`, skip collection (the player just walks through it).

### 4. Drawing Logic
- In `draw()`, render big dots larger:
  - Standard dot: radius 6, yellow `#f3ca20`.
  - Big dot: radius 12, orange `#f39c12` (or glowing ring) so it stands out visually.
  - We can draw an outer pulsing glow ring for big dots to emphasize their status.

```javascript
  // Draw Dots
  for (const dot of dots) {
    if (!dot.collected) {
      const isBig = !!dot.big;
      const dotRad = isBig ? 12 : 6;
      ctx.fillStyle = isBig ? '#f39c12' : '#f3ca20';
      ctx.shadowBlur = isBig ? 14 : 8;
      ctx.shadowColor = ctx.fillStyle;
      ctx.beginPath();
      ctx.arc(dot.x, dot.y, dotRad, 0, Math.PI * 2);
      ctx.fill();
      ctx.shadowBlur = 0; // reset
    }
  }
```
